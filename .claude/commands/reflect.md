このセッションでのやりとりを振り返り、CLAUDE.md の改善点を特定してください。

実行手順:

1. このセッションでユーザーから受けた指摘、依頼、修正依頼を振り返る
2. それらを抽象化し、今後の一般的なケースに適用できる学びを抽出する
3. 以下の観点で CLAUDE.md への追加・修正を提案する:
   - Claude Code の役割と境界: 学習サポートとしての立ち位置の明確化
   - インタラクション方針: ユーザーとのコミュニケーションのあり方
   - 説明の粒度: 説明の詳しさやレベル感
   - 回答のフォーマット: コード例、箇条書き、見出しなどの構造化
   - 技術選択: どのツール、機能、パターンを優先すべきか
   - 禁止事項: 避けるべき行動パターン

注意事項:
- セッション特有の具体的な情報 (ファイル名、変数名など) は含めない
- 他のプロジェクトでも応用可能な抽象度の高い指針にする
- 既存の CLAUDE.md の内容と矛盾しないようにする
- 太字や斜体などの過度な装飾を使わない
- 冗長な表現を避ける

出力形式:
- 追加すべきセクションとその内容
- 既存セクションへの追記案
- 必要に応じて削除・修正すべき内容
