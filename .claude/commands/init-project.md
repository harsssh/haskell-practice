# 新しい Haskell 学習プロジェクトの初期化

このコマンドは、会話履歴で決定したテーマから Cabal プロジェクトを自動生成します。

## 実行手順

1. 会話履歴からテーマを特定し、適切なディレクトリ名 (kebab-case) を決定してください

2. ディレクトリが既に存在しないか確認し、存在する場合はエラーを表示して終了してください

3. 以下の手順でプロジェクトを作成してください:

   a. ディレクトリを作成し、`cabal init` を実行
      - `--non-interactive`: 対話モードを無効化
      - `--libandexe`: library と executable の両方を生成
      - `--tests`: test suite を生成
      - `--license=MIT`: fizzbuzz と同じライセンス
      - `--language=GHC2021`: 最新の言語標準

   b. `cabal.project` ファイルを生成 (fizzbuzz と同じ設定):
      ```
      packages: ./*.cabal

      test-show-details: direct
      ```

4. テーマに基づいて README.md を生成してください。以下の構成で記述:

   ```
   # <テーマ名>

   ## 概要
   - プロジェクトの説明 (2-3 文)
   - どのような場面で使われる技術か

   ## 実装する機能
   - 基本機能の箇条書き (3-5 項目)

   ## 学習目標
   このプロジェクトを通じて学ぶ Haskell の概念やテクニック:
   - 言語機能 (例: 代数的データ型、パターンマッチ、高階関数)
   - 型システムの活用 (例: newtype, 型クラス)
   - 推奨ライブラリ
   - テストの書き方

   ## 技術的なヒント
   - 実装アプローチ (コードは含めない)
   - 推奨ライブラリと用途 (Context7 で最新情報を取得)
   - 注意点や落とし穴

   ## 参考資料
   - 関連ドキュメントやリソース
   ```

   重要な制約:
   - コード実装は含めない (CLAUDE.md の方針に従う)
   - 学習者が自分で実装できるレベルのヒント
   - Context7 でライブラリの最新情報を調査
